HTML Timing?

Ever wanted to…

  • create your own slideshow in HTML?
  • synchronize HTML content with media streams? (subtitles, transcriptions, annotations…)
  • animate HTML elements?

…without writing any single line of JavaScript?

SMIL Timing & Timesheets

  • a declarative standard — W3C recommendation
    (= one of the 12 SMIL modules)
  • designed for XML element synchronization
    (works fine with HTML and SVG, too)
timeContainer = "par|seq|excl"
specify a temporal behaviour to any HTML/SVG container
timeAction = "display|visibility|class:[className]|style|intrinsic"
defines how an element is to be activated / unactivated

“HTML Timing” Project

a.k.a. the “two-step world domination plan”

  1. SMIL Timing implementation
    (MIT license — less than 10 KB of JavaScript)
    • to “animate” HTML elements
    • to synchronize HTML5 <audio|video> media
    • to design slideshows, web documentaries, multimedia presentations…
  2. development of a wysiwyg editor
    (GPL/LGPL/MPL license — based on Mozilla Composer)
  3. possible SVG2 / HTML5 proposal

timeContainer="seq"

A seq container defines a sequence of elements in which elements play one after the other.

  1. first item
  2. second item
  3. third item
  4. fourth item
  5. last item 


    

      
timeContainer="par"

      

        A par container, short for "parallel", defines a simple time grouping in
        which multiple elements may play back at the same time.

timeContainer="excl"

      

        Similar to par, but only one child element may play at any given time.

        If any element begins playing while another is already playing,
        the element that was playing is stopped.

 Slideshows 

      

        A <seq> time container is a good basis for a
        slideshow engine:
      

      
    
        
  • 
          each slide is a child of this main <seq> time
          container
        
    • 
        
  • 
          the first|prev|next|last attributes provide a navigation
          menu "out of the box"
        
    • 
        
  • 
          setting an ID attribute to each slide allows to use hyperlinks to get
          a direct access
        
    • 
      

    

    


    

      
 Progressive Display 

      
    
        
  •  On this slide 
    • 
        
  •  the list items 
    • 
        
  •  appear one after another 
    • 
        
  •  at 2s intervals. 
    • 
      

      

        The next slide will appear in 10s or if you click
        on the "Next" button below — whatever comes first.
      

    

    


    

      
 Incremental Display 

      
    
        
  •  Click on this slide 
    • 
        
  •  and the list items  
    • 
        
  •  will appear         
    • 
        
  •  one after another.  
    • 
      

      

        (you still have to click the "next" button to get the next slide)

 Key Benefits 

      
    
        
  • 
          content / presentation=CSS / animation=SMIL 
    
          (check the “Timesheets” section on our website)
        
    • 
        
  • 
          declarative management of user interactions
    
           ⇒ little to no JavaScript code to write 
        
    • 
        
  • 
          extensible using DOM events (onbegin/onend)
        
    • 
        
  • 
          stable syntax: W3C recommendation
    
          (maybe in SVG2 or HTML5 some day?)
        
    • 
        
  • 
          lightweight JavaScript library (<10 KB) 
    
          cross-browser, no jQuery/YUI/whatever dependency
